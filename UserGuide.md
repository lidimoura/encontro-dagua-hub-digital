# 📘 UserGuide — Encontro d'Água Hub CRM

> Complete manual of all native functionalities and implemented features.

---

## 🏠 System Overview

**Encontro d'Água Hub** is a modular SaaS CRM featuring:
- **Opportunities Kanban** with multiple boards and drag-and-drop
- **Embedded AI Team** in cards (Precy, Jury, AIFlow, AI Hub)  
- **Contact Management** with full lifecycle (Lead → Customer)
- **Product Catalog** separated from internal Tech Stack
- **Automatic Briefing** via SDR (Amazô) capturing leads from the Landing Page

---

## 1. Kanban (Boards)

### Creating a Board with AI
1. Go to **Boards** → click the **+** button in the board selector
2. Describe the board's objective (e.g., "Sales pipeline for international clients")
3. AI automatically generates appropriate stages and context configurations

### Available Templates
| Template | Included Stages | Included AI Agents |
|---|---|---|
| **SDR Pipeline** | Cold List → Scheduled → Proposal → Closed | Precy, Jury, WA AI |
| **Onboarding** | Contracted → Kit Sent → Active → Completed | Jury, AIFlow |
| **Projects** | Backlog → In Progress → Review → Delivered | AIFlow, Precy |
| **Custom** | Defined by user | All available |

### Opportunities `+` Button
- Appears in each Kanban column
- Creates a new deal directly in the corresponding stage
- Quick form: Title, Value, Contact (links or creates)

### Drag and Drop (DnD)
- Drag cards between columns to change stages
- Status is updated in Supabase and synchronized in real-time
- Opened Card reflects the change instantly (Board ↔ Card sync)

## 2. Lead Card (DealDetailModal)

### Available Tabs
| Tab | Content |
|---|---|
| **Overview** | General data, value, tags, custom fields |
| **Briefing** | Automatic message captured by Amazô SDR |
| **Contact** | Lead data + WhatsApp AI button |
| **Products** | Services added to the deal + declared interests |
| **Timeline** | Activity history and notes |
| **Documents** | Contracts generated by Jury |
| **AI Team** | Precy, AIFlow, Jury — agents inside the card |

### WhatsApp + AI Message
- Appears automatically when the lead has a registered phone number
- **📲 WhatsApp + AI Msg** button: generates personalized message via Gemini
- Message is editable before sending
- Opens pre-filled `wa.me/{number}?text=...`
- Requires: AI API Key configured in Settings

### Closing Flow
1. Progress bar status (top of card) — click to move stages
2. **WON** button (green): moves to `CLOSED_WON`
3. **LOST** button (red): moves to `CLOSED_LOST`
4. **CONVERT** button (teal): promotes contact to `CUSTOMER` in lifecycle

## 4. Contact Management

### Lifecycle Stages
```
LEAD → MQL → PROSPECT → CUSTOMER
```
- Editable stages in **Settings → Lifecycle**
- Customizable names and colors

### Duplicate Contacts (Djavan Case)
- The system automatically filters contacts without a valid `contactId` when rendering the Kanban
- Cards without a contact display a "NO CONTACT" badge — can be linked manually in the card
- Use **Link to existing contact** or **+ Create new contact** inside the card

- Shows only products `is_internal: false` and `product_type != 'tech_stack'`
- Precy's products appear here after "Save to Catalog"
- Dropdown inside cards (Products tab) uses the same filtering — never shows internal tools

## 6. AI Settings

### API Keys (Automatic Failover)
1. Go to **Settings → AI**
2. **Main Key**: your primary API key (Gemini, OpenAI, or Claude)
3. **Backup Key (429 Failover)**: automatically takes over when main key hits quota
4. Click **Save** — keys are persisted in Supabase (not dependent on `.env` or Vercel)

### Available Models by Provider
| Provider | Models | Note |
|---|---|---|
| Google Gemini | 2.5 Flash, Flash Lite, 2.5 Pro, 3.0 Pro | Flash = recommended |
| Anthropic Claude | Sonnet 4.5, Haiku 4.5, Opus 4.5 | Sonnet = recommended |
| OpenAI | GPT-4o, GPT-4 Turbo, GPT-3.5 Turbo | — |

### Advanced Settings
- **Thinking (Google)**: enables chain-of-thought reasoning before answering
- **Prompt Caching (Anthropic)**: saves tokens in long conversations
- **Web Search**: connects the model to the internet (Google Grounding / Anthropic tools)

## 8. Lead Automations and Real-Time (No F5)

The CRM supports **3 main Lead entry points**, and all of them trigger real-time Kanban updates (no F5 refresh needed):

### 1. Link d'Água (NFC / QR Code)
- **Flow:** Visitor scans NFC/QR Code → Accesses Bridge profile → Clicks WhatsApp / Action button.
- **Automation:** If configured via external N8N/Webhook, interaction data capture creates the Lead in CRM.

### 2. Amazô-SDR (Landing Page)
- **Flow:** Visitor fills conversational form / Typebot on LP → Data sent to Supabase.
- **Automation:** The `amazo-sdr` (or `process-lead`) Edge Function processes the payload, finds/creates `Contact` (preventing duplicates if phone exists), and creates an automatic `Deal` in the first stage of the configured board (`LEAD` status).
- **Briefing:** Provided data (name, services, company, phone) is saved in the modal's **Briefing** tab, allowing human rep or AI to start the conversation (via WhatsApp AI button) with full context.

### 3. Manual Creation (Kanban UI)
- **Flow:** User clicks the **`+` Button** located in any Kanban column header.
- **Action:** Opens the quick form to register Lead Name, Value, and select an existing Contact (or create one on the fly).

### ⚡ The Real-Time Kanban "Brain"
- Regardless of which of the 3 routes creates the Lead, you **don't need to reload the tab**.
- The system keeps an active WebSockets connection (`postgres_changes`) listening to the `deals` table.
- The Kanban automatically reloads the visual cards in the right stage (in about 1 to 2 seconds) as soon as the database insertion occurs.

### AI Settings — Dual Key (Failover)
- **Main Key**: used in all normal AI calls.
- **Key Note**: free text field to identify origin (e.g., "Personal Google Account").
- **Backup Key**: takes over automatically if the main key usage limit is reached (429 quota error).
- **Security and DB**: Keys saved in Supabase itself (`user_settings`) with an "Eye" button to view in the UI.

```sql
    ADD COLUMN IF NOT EXISTS ai_api_key_secondary text,
    ADD COLUMN IF NOT EXISTS ai_api_key_note text,
    ADD COLUMN IF NOT EXISTS ai_api_key_secondary_note text;
```
